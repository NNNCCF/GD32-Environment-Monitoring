# GD32 FreeRTOS 环境检测系统

## 项目简介

本项目基于 **GD32F10x + FreeRTOS**，实现多任务环境检测与状态显示，集成以下模块：

- DHT11 温湿度传感器
- MQ2 气体传感器（ADC 采样）
- OLED 显示屏（中文界面）
- 按键翻页
- LED 报警指示
- UART 串口输出与命令控制

## 主要功能

- FreeRTOS 多任务调度
- 温湿度采集与阈值报警
- MQ2 气体浓度百分比显示
- MQ2 电压换算显示（单位：`mV`）
- OLED 三页面中文界面
- 串口周期输出传感器数据
- 看门狗健康检查与自动复位保护

## 报警阈值

- 温度：`> 30°C` 报警（同时保留低温阈值 `< 10°C`）
- 湿度：`> 60%` 报警（同时保留低湿阈值 `< 30%`）
- 气体浓度：`> 40%` 报警

## OLED 界面说明

通过按键切换页面：

1. 温湿数据页
   - 温度
   - 湿度
2. 气体数据页
   - 浓度百分比
   - 原始电压（由 ADC 原始值换算，单位 `mV`）
   - 平均电压（滑动平均后换算，单位 `mV`）
3. 报警信息页
   - 温度状态（正常/报警）
   - 湿度状态（正常/报警）
   - 气体状态（正常/报警）+ 当前浓度百分比

## MQ2 电压换算

项目中使用 12bit ADC（0~4095）和 3.3V 参考电压：

```c
mV = ADC * 3300 / 4095
```

- `原始电压`：使用当前原始采样值换算
- `平均电压`：使用滑动平均后的 ADC 值换算

## 串口输出

- 波特率：`115200`
- 格式：`8N1`

示例输出：

- `MQ2: 45%, 1.482V`
- `DHT11: Temp=26 C, Humi=58%`

## 串口命令

- `LED:ON`
- `LED:OFF`
- `LED:TOGGLE`

## 任务说明

- `vTaskMQ2`：采集 MQ2，计算百分比与电压并串口输出
- `vTaskDHT11`：采集温湿度并计算报警状态
- `vTaskOLED`：刷新 OLED 中文界面
- `vTaskLED`：报警状态指示
- `vTaskKEY`：按键扫描与页面切换
- `vTaskUartCmd`：处理串口命令
- `vTaskWDG`：监控关键任务心跳并喂狗

说明：系统总报警由三项共同决定，任一触发即进入报警状态并点亮 LED：

- 温度报警
- 湿度报警
- 气体浓度报警

## 看门狗策略

系统并非“固定周期无条件喂狗”，而是：

- 关键任务按周期上报心跳
- 仅在所有关键任务均健康时喂狗
- 若任务卡死/阻塞超时，停止喂狗并由硬件复位

## 硬件连接（默认）

- DHT11 DATA -> `PB8`
- MQ2 AO -> `PA0`
- OLED SCL -> `PB6`
- OLED SDA -> `PB7`
- LED -> `PC13`
- KEY -> `PE6`
- UART TX -> `PA9`
- UART RX -> `PA10`

## 构建与下载

1. 打开 `USER/test.uvprojx`
2. 选择对应 GD32 芯片型号
3. 配置下载器（J-Link / ST-Link）
4. 编译并下载运行
